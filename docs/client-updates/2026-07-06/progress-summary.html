What I've built so far

The first stage is done: the MVP is built and it's looking really good. Right now it's being tested against some documents from last quarter, and I've already run it through an initial pilot, which came back solid.

What it can do today

  • Reads both PDF documents and web pages.
  • Handles long documents by splitting them into sections without losing the thread between them.
  • Takes page snapshots and reads tables, images, and basic charts, not just plain text. Many allocation views live in dial graphics and tables rather than sentences, so this matters.
  • Was given a "financial analyst brain": worked examples and house conventions built from the sample documents in your earlier email, so it reasons like an analyst rather than a generic AI.
  • Produces one output spreadsheet per run in the agreed format: firm, date, source, sub-asset class, the call, and the exact supporting text with a page reference for every call.

How I keep it trustworthy

  • Every call must quote the exact source text and page that supports it. Calls that can't be verified against the source are rejected rather than kept.
  • A second, independent AI reviewer re-reads every call against the original source before it is accepted. Think of it as a built-in checker.
  • Dial and chart graphics are verified by that independent reviewer looking at the page image itself, not just the page text, so allocation views published as dials are read reliably.
  • Confidence is not the AI grading itself. It comes from a fixed, transparent set of rules (was the quote verified? did the checker agree? how direct was the language?), with the AI only supplying the raw ingredients. Anything low-confidence is automatically flagged for human review.
  • Where the evidence is genuinely weak or unclear, the system says "uncertain" instead of guessing.
  • Every run also produces a separate failures file alongside the results: calls the system attempted but rejected, for example because the supporting quote could not be verified, or the checker disagreed, each with the reason. This keeps rejected calls reviewable on their own, and keeps them distinct from "uncertain" calls. "Uncertain" means the document itself was ambiguous; a "failure" means the system found something but couldn't stand behind it.
